- Mevcut çalışan authentication yapısını değiştirme

Amaç:
Mevcut Enerji Yönetim Sistemi uygulamasına “Enerji Kullanım Grupları” yapısını ekle. Bu yapı sayaçların ISO 50001 analiz amacıyla gruplandırılması için kullanılacak.

Mevcut yapı:

- Birim Yönetimi ekranında birimler ve alt birimler / lokasyonlar tanımlanıyor.
- Sayaç Yönetimi ekranında sayaçlar tanımlanıyor ve alt birime bağlanabiliyor.
- Tüketim Verileri ekranında sayaçlara ait aylık tüketimler giriliyor.
- Bu yapı korunacak.
- Yeni yapı mevcut yapıya ek olarak çalışacak.

İstenen yeni mantık:
Birim / Alt Birim = Tüketim nerede gerçekleşiyor?
Sayaç = Tüketim nasıl ölçülüyor?
Enerji Kullanım Grubu = Tüketim hangi sistem, proses, faaliyet veya analiz başlığına ait?

Örnek enerji kullanım grupları:

- Basınçlı Hava Sistemi
- Kazan Sistemi
- Araç Yakıtları
- Üretim Elektriği
- Chiller Sistemi
- İdari Bina Enerjisi
- Kompresör Sistemi
- HVAC Sistemi

1. Veritabanı değişikliği:

Drizzle schema içinde yeni bir tablo oluştur:

energyUseGroups

Alanlar:

- id
- companyId: zorunlu, multi-tenant veri ayrımı için
- name: grup adı
- code: kısa kod, opsiyonel ama benzersiz olmalı
- groupType: production, building, utility, vehicle, process, hvac, lighting, other
- energySourceId: nullable
- unitId: nullable
- subUnitId: nullable
- description: nullable
- isSeuCandidate: boolean, default false
- isActive: boolean, default true
- createdBy
- createdAt
- updatedAt

Kurallar:

- Aynı companyId altında aynı name ile mükerrer aktif grup oluşmasın.
- Tüm sorgularda companyId filtresi zorunlu olsun.
- Başka şirkete ait grup görüntülenmesin, güncellenmesin, silinmesin.

Mevcut meters / sayaç tablosuna şu alanı ekle:

- energyUseGroupId: nullable foreign key

Bu alan opsiyonel olmalı. Mevcut sayaç kayıtlarını bozma.

Migration veya drizzle push çalıştırma. Sadece schema değişikliğini hazırla ve yapılması gereken komutu raporla.

2. Backend REST API:

Aşağıdaki endpointleri ekle:

GET /api/energy-use-groups

- Sadece kullanıcının companyId değerine ait aktif/pasif grupları döndürsün.
- Filtreler desteklensin:
  - isActive
  - groupType
  - energySourceId
  - unitId
  - subUnitId

POST /api/energy-use-groups

- Yeni enerji kullanım grubu oluştursun.
- companyId kullanıcı oturumundan alınsın, frontend’den gelen companyId’ye güvenilmesin.
- name zorunlu olsun.
- groupType zorunlu olsun.
- createdBy kullanıcıdan alınsın.

PUT /api/energy-use-groups/

- Sadece aynı companyId içindeki kayıt güncellensin.
- Başka şirket kaydı bulunursa 404 veya yetkisiz cevap dönsün.

PATCH /api/energy-use-groups//status

- Aktif/pasif durumunu değiştirsin.
- Fiziksel silme yapma, pasifleştir.

GET /api/energy-use-groups//meters

- Bu gruba bağlı sayaçları listele.
- Sadece kullanıcının companyId değerine ait sayaçları döndür.

Mevcut sayaç oluşturma ve güncelleme endpointlerine energyUseGroupId desteği ekle:

- Sayaç oluştururken enerji kullanım grubu seçilebilsin.
- Sayaç güncellenirken enerji kullanım grubu değiştirilebilsin.
- energyUseGroupId boş/null olabilir.
- Seçilen energyUseGroupId mutlaka aynı companyId içinde var olmalı.
