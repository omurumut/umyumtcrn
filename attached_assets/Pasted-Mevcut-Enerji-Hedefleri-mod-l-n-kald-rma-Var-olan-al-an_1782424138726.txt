Mevcut “Enerji Hedefleri” modülünü kaldırma. Var olan çalışan yapıyı analiz ederek ISO 50001’e uygun “Enerji Amaçları, Hedefleri ve Eylem Planları” modülüne modernize et.

Ayrıca Verimlilik Artırıcı Projeler, yani VAP yapısını da bu modülle bağlantılı olacak şekilde hazırla.

ÖNEMLİ KARAR:
- Her eylem planı VAP değildir.
- Ancak her VAP bir eylem planından türemelidir.
- Bu nedenle VAP ayrı menü/modül olarak görünsün ama veri ilişkisi eylem planına bağlı olsun.
- Eylem planı içinde “Bu eylem VAP mı?” alanı olsun.
- is_vap = true olan eylemler ayrıca Verimlilik Artırıcı Projeler modülünde listelensin.
- VAP detayları ayrı tabloda tutulabilsin.

Genel kurallar:
- Mevcut Enerji Hedefleri modülünü silme.
- Mevcut hedef kayıtlarını silme.
- Mevcut tabloyu drop etme.
- Gerekirse mevcut hedef tablosuna migration ile yeni kolonlar ekle.
- Multi-tenant company_id izolasyonunu bozma.
- Mevcut auth ve role yapısını kullan.
- Firma admini tüm birimleri görebilsin.
- Normal kullanıcı sadece kendi birimine ait kayıtları görebilsin.
- Mevcut tasarım dilini koru.
- Minimum ve kontrollü değişiklik yap.
- GitHub kaynak yapısını bozma.

Mevcut Enerji Hedefleri ekranını şu yapıya dönüştür:

2. Eylem Planları

4. VAP bağlantısı

energy_action_plans

- status

- notes nullable

Eylem planı durumları:
- planned = Planlandı
- in_progress = Devam Ediyor
- completed = Tamamlandı
- delayed = Gecikti
- cancelled = İptal

Öncelik seçenekleri:
- low = Düşük
- medium = Orta
- high = Yüksek

Eylem Planları sekmesinde:
- Hedef seçilebilsin.
- Seçilen hedefe bağlı eylemler listelensin.
- Yeni eylem ekleme/düzenleme/silme olsun.
- Formda “Bu eylem VAP mı?” checkbox alanı olsun.
- progress_percent 0 ile 100 arasında validasyonlu olsun.
- investment_cost ve expected_cost_saving varsa payback_months otomatik hesaplanabilsin:
  payback_months = investment_cost / (expected_cost_saving / 12)
- Kullanıcı isterse geri ödeme süresini manuel değiştirebilsin.

Eylem planı listesi kolonları:
- Hedef
- Eylem adı
- Sorumlu

- Yatırım maliyeti
- Geri ödeme süresi
- VAP mı?
- Başlangıç
- Bitiş
- İlerleme
- Durum
- İşlemler

İZLEME VE GERÇEKLEŞME:
Yeni tablo oluştur:

energy_target_progress veya energy_objective_progress

Alanlar:
- id
- company_id
- target_id veya objective_id: mevcut enerji hedefi tablosundaki id’ye FK
- period_year
- period_month nullable
- actual_value numeric
- actual_saving_value numeric nullable
- comment nullable
- recorded_by
- recorded_at

Bu sekmede:
- Hedef seçilebilsin.
- Seçilen hedefin baz değeri, hedef değeri, gerçekleşen değeri ve sapması gösterilsin.
- Progress kayıtları listelensin.
- Yeni gerçekleşme kaydı eklenebilsin.
- Son progress kaydı, hedefin actual_value/gerçekleşen değer alanını güncelleyebilsin.
- Azaltım hedeflerinde:
  - actual_value <= target_value ise “Hedef sağlandı”
  - actual_value > target_value ise “Riskli / hedef altında”
- Eğer baz değer veya hedef değer yoksa hesaplama yapma.
- Kullanıcıya “Baz değer ve hedef değer girildiğinde gerçekleşme durumu hesaplanır.” mesajı göster.

API:
Mevcut enerji hedefleri API endpointlerini bozma.
Gerekirse genişlet.

Yeni endpointler:

Eylem planları:
- GET /api/energy-action-plans
- POST /api/energy-action-plans
- PUT /api/energy-action-plans/:id
- DELETE /api/energy-action-plans/:id

İzleme:
- GET /api/energy-target-progress
- POST /api/energy-target-progress
- DELETE /api/energy-target-progress/:id

VAP:
- GET /api/vap-projects
- POST /api/vap-projects
- PUT /api/vap-projects/:id
- DELETE /api/vap-projects/:id

API kuralları:
- Auth zorunlu.
- Tüm sorgularda company_id filtresi zorunlu.
- Normal kullanıcı sadece kendi unit_id kayıtlarını görebilsin.
- Admin tüm birimlerin kayıtlarını görebilsin.
- Action plan eklenirken bağlı hedefin aynı company_id içinde olduğu doğrulansın.
- Progress eklenirken bağlı hedefin aynı company_id içinde olduğu doğrulansın.
- VAP detayı eklenirken bağlı action_plan aynı company_id içinde doğrulansın.
- VAP detayı eklenirken action_plan.is_vap = true olmalı.
- VAP listesinde bağlı action plan, bağlı hedef, birim ve enerji kaynağı bilgileri join ile gösterilsin.
- Hata mesajları frontend’de kullanıcıya gösterilsin.
